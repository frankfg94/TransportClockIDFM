# Internationalization Guide

Transport Clock uses a small typed i18n layer under `src/i18n`.

## Golden Rule

Do not add user-facing raw text directly in Vue templates, composables, API payloads consumed by the UI, or feature modules.

Use translation keys instead:

```vue
<script setup lang="ts">
import { useI18n } from "../i18n";

const { t } = useI18n();
</script>

<template>
  <button>{{ t("common.actions.save") }}</button>
</template>
```

## Files

- `src/i18n/types.ts` defines supported locales and language preferences.
- `src/i18n/i18n.ts` exposes `useI18n`, `translate`, `resolveLocale`, `d`, and `n`.
- `src/i18n/messages/fr.ts` contains the French catalog.
- `src/i18n/messages/en.ts` contains the English catalog.
- `src/i18n/messages/schema.ts` derives the shared catalog shape from French and allows English to use different string values.

## Add a Translation

1. Pick a stable key that matches the feature area, for example `settings.display.weather`.
2. Add the French string in `src/i18n/messages/fr.ts`.
3. Add the English string in `src/i18n/messages/en.ts`.
4. Use `t("settings.display.weather")` in the component.
5. Run `npm.cmd run tsc` so TypeScript verifies both catalogs have the same shape.

With parameters:

```ts
t("traffic.groupLineOther", { count: lines.length });
```

Catalog entry:

```ts
groupLineOther: "{count} lines";
```

## Dates and Numbers

Use the helpers returned by `useI18n`:

```ts
const { d, n } = useI18n();

d(new Date(), { hour: "2-digit", minute: "2-digit" });
n(42, { maximumFractionDigits: 0 });
```

These helpers automatically use `fr-FR` for French and `en-US` for English.

## Language Preference

The persisted app setting is `settings.language`:

- `"auto"` keeps the default behavior.
- `"fr"` forces French.
- `"en"` forces English.

Automatic mode resolves to French only when the browser timezone is exactly `Europe/Paris`; otherwise it resolves to English. Users can change the language from `pages/settings.vue`.

## Tests

Language behavior is covered by:

- `tests/i18n.test.ts` for locale resolution, translation, and interpolation.
- `tests/settingsPage.dom.test.ts` for the Settings UI language switch.
- `tests/appSettings.test.ts` for persistence defaults and normalization.

When adding a feature with meaningful translated UI, add either a unit test for the formatter or a DOM assertion that proves the translated string appears through `t`.

## AI Contributor Checklist

Before finishing a UI change:

1. Search for new raw user-facing strings in changed Vue/TS files.
2. Move them into both catalogs.
3. Use `t`, `d`, or `n` instead of inline text formatting.
4. Keep code comments in English.
5. Run `npm.cmd run tsc`.
